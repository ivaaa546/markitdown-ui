# progress/

Este directorio contiene el **registro de avance** de cada feature completada.

## Convención

Por cada spec que pasa a estado `DONE`, el Leader crea un archivo:

```
progress/<slug>.md
```

donde `<slug>` coincide con el slug del archivo de spec correspondiente
(sin la fecha).

## Formato

Ver la sección "Registro de progreso en `progress/`" en
[docs/verification.md](../docs/verification.md) para el formato completo.

## Estado actual

> No hay features completadas aún. Este directorio se llenará a medida
> que las specs avancen hasta `DONE`.
